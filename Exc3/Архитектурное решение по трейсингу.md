# Мотивация
Добавление системы трейсинг поможет компании понимать о том, в каком месте, и по какой причине заказы могут пропадать или задерживаться на всех этапах обработки. Внедрение данного решения критически необходимо для быстрого обнаружения и устранения проблем. Внедрение системы трейсинга в конечном итоге приведет к росту удовлетворённости клиентов, увеличению числа заказов, росту прибыли компании.

Технические и бизнес-метрики решения, на которые повлияет внедрение трейсинга:
1. Number of HTTP 500 for API - ошибки будут более оперативно исправляться, сокращение среднего количества ошибок, снижается время решения проблемы
2. Number of dead-letter-exchange letters in RabbitMQ - исправление ошибок также приведёт к сокращению числа потерянных сообщений в RabbitMQ, снижается время решения проблемы
3. Response time (latency) for API - выявленные "узких горлышек" в системе поможет точечно масштабировать и экономить деньги на инфраструктуре, а так же снижается время решения проблемы
4. Время обработки заказа
5. Время загрузки и обработки 3D-моделей

# Предлагаемое решение
Предлагается реализовать решение используя следующие технологии:
- OpenTelemetry - передача данных телеметрии в Jaeger
- Jaeger - удобный инструмент и для визуализации трассировок в распределённых системах.
- Jaeger UI - веб-интерфейс для поиска и просмотров трейсов.

# Компромиссы
Трассировка принесет больше пользы для MES API и CRM API т.к. в этой части были замечены проблемы с заказами. Добавление трассировки в SHOP API можно отложить на более поздний срок.

# Аспекты безопасности
Для предотвращения несанкционированного доступа, необходимо настроить аутентификацию и авторизацию, предоставиться доступ только администраторам и разработчикам.
Использовать шифрованные каналы связи.
Данные трейсинга должны иметь минимально необходимую информацию.
